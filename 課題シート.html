⭐ 課題A：単語生成エージェントを作る ⭐

所要：10分 / Copilot Agent / 編集ポイント ①

やること

Copilot Agent で、テーマを投げると単語10個＋ヒントを返してくれるエージェントを作る。
最初の指示はあえてシンプルに。たぶん出力はバラバラで貼り付けに苦労します。（次の B で整えます）。

手順

  1. チームでテーマを決める（例: 動物、アニメ、プログラミング用語、社内用語 など）
  2. Copilot Agent で新しいエージェントを作る。エージェントへの指示（instructions / システムプロンプト）に、まずは最小限だけ書いてみる： 
    あなたはクロスワード用の単語生成アシスタントです。
ユーザーが指定したテーマについて、
単語を10個、ヒントとセットで考えてください。
  3. エージェントにテーマ（例：「動物」）を投げて、出力を見てみる。
    たぶんこんな感じになるはず:
    • マークダウンの箇条書きや表形式で返ってきて、そのままでは貼れない
    • 単語がカタカナ・ひらがな・漢字で混ざっている
    • 「以下が動物の単語10個です…」みたいな前置きが入る
  4. 出力を 手で整形しながら crossword.html の ① にある WORDS_AND_HINTS に貼り付けてみる。THEME もテーマ名に変える。
  5. ファイルを保存して、ブラウザのタブで Ctrl+R（更新）。画面上部の「配置結果 X / Y 語」を確認。
整形に時間をかけすぎないのがコツ。貼れる範囲で貼って配置数を1回見たら先へ進みましょう。次の B-1 で「整形そのものをエージェントに任せる」改良をします。
うまく貼れなくて画面が真っ白になっても大丈夫。F12（DevTools）→ Console でエラーの場所が分かります。配列の [ ]、各要素のカンマ , が崩れやすいポイント。どうしても動かなければそのまま B-1 に進んでOK。

⭐⭐ 課題B：エージェントの指示を4フェーズで磨く ⭐⭐

所要：40分（4フェーズ × 10分）/ Copilot Agent / 編集ポイント ①

やること

課題Aで作ったエージェントは配置数が少なめなはず。4つのフェーズに分けて段階的に指示を改良していきます。
各フェーズ後に配置数を記録して、「どの改良がどれだけ効いたか」を可視化しましょう。

なぜ配置数が変わるのか

クロスワードは「単語同士が共通の文字で交差する」ことで配置されます。共通文字を持つ単語ペアが少ないと、どんなにアルゴリズムを頑張っても配置できません。つまり 「共通文字が多い単語セット」を出してくれるエージェントが正解に近い。

🅱️1 出力を「そのまま貼り付けられる形」に揃える（10分）

課題Aで「毎回手で整形するのしんどい…」となったはず。ここを解決する。
エージェントの指示に次の2つを追加:

  • 出力フォーマットを指定：下記のようなJavaScript配列リテラル形式だけで出してもらう。 
    出力は以下のJavaScript配列リテラル形式のみで返してください。
前置き・後書きの説明文・マークダウンのコードフェンスは一切含めないでください。

[
  {word: "ラーメン", hint: "中華麺を使った料理"},
  {word: "カレー",   hint: "スパイスの効いた料理"}
]
    → crossword.html の WORDS_AND_HINTS にそのままコピペできる形で出してもらう、という発想。整形作業をエージェント側に寄せる。
  • 使う文字種を絞る：「単語はカタカナかひらがなで書く。漢字・英字・数字は使わない」 → クロスワードは「文字の一致」で交差する。漢字や英字が混ざると交差マスを作れない。
    （このアプリではカタカナとひらがなを同じ文字として扱うので、どちらか一方でも混在でもOK）
指示を更新 → エージェントを再実行 → WORDS_AND_HINTS に貼り付け（今度はコピペだけで済むはず）→ 配置数を記録

🅱️2 共通文字を意識させる（10分）

ここが本日の肝。「共通文字が多い単語を選べ」とエージェントに考えさせる。次を試す:

  • 明示的に要求：「単語同士で共通する文字が多くなるように選ぶこと」
  • 2段階で考えさせる：「まず候補を30個挙げる → その中から共通文字が最も多くなる10個を選んで出力」
  • 具体例（few-shot）：「例: 『ラーメン』と『テンプラ』は『ラ』『ン』が共通、『ラーメン』と『ウドン』は『ン』が共通。こうやって文字がつながる関係を多く含むセットにする」
  • NG例を示す：「NG例: 『トラ』『イヌ』『ゾウ』のように共通文字が1つもない組み合わせは避ける」
効果が大きいのはどれか？複数入れるとどうなる？試行錯誤してみましょう。

🅱️3 自由工夫で最高記録を狙う（10分）

ここまでで学んだことを総動員。チームで自由に工夫して配置数を伸ばす。アイデア例:

  • 「共通文字が多いペアが最低5組できるように」など数値的な制約を足す
  • 頻出の文字を多く含む単語を優先するよう指示
  • エージェントへの「役割」をより具体化（例：「あなたはクロスワードパズル作家で、30年の経験がある」）
  • 編集ポイント ② の SEED の数字を変えて別レイアウトを試す（同じ単語でも配置数が変わることがある）
指示を盛りすぎて品質が下がることもあります（ヒントがテキトーになる、単語がマイナーすぎる等）。効かなかった一文は消すことも大事。

🅱️4 別テーマで汎用性を検証する（10分）

B-3まででベストなエージェントができたはず。でも、それは最初のテーマ専用に「過剰チューニング」されているかも。全く違うテーマを投げて、同じくらい配置数が出るかテストします。

  1. 元と大きく違うテーマを選ぶ（例: 元が「動物」なら「楽器」「都道府県」「プログラミング言語」など）
  2. 指示は変えずに、同じエージェントに新しいテーマを投げる
  3. 出力を WORDS_AND_HINTS に貼り付けて配置数を確認
  4. 比較：
    • ✅ 元のテーマと同じくらい配置できた → 汎用性の高い良いエージェント
    • ⚠️ 大幅に落ちた → 元のテーマに過剰チューニング。指示が具体的すぎた可能性
  5. もし落ちていたら、指示を「どんなテーマでも使える」形に書き直して再チャレンジ
💡 学びのポイント：良いプロンプトは、入力が変わっても同じ品質を保てるもの。特定の入力だけに効く指示は、実は弱い指示。実務のエージェント設計でも効いてくる感覚です。

🎮 クロスワードの操作方法

  • マスをクリックして文字入力 → 自動で次のマスへ進む
  • 矢印キー（← → ↑ ↓）で移動
  • Backspace で前のマスへ戻って消去
  • Space または同じマスを2回クリックで縦↔横を切り替え
  • ヒント文をクリックするとその単語の先頭マスに飛ぶ
  • ひらがな・カタカナどちらで入力してもOK（自動でカタカナに変換）
  • 「答え合わせ」で正解マスが緑、間違いマスが赤に
  • 「全部見る」で答えが全マス表示される（時間切れの時用）

🆘 困ったとき

  • 画面が真っ白になった → JavaScriptのエラー。配列のカッコ・カンマが崩れていないか確認。F12（DevTools）→ Consoleタブにエラーが出ている
  • 配置数が増えない → 単語の表記揺れ（カタカナ/ひらがな混在）を疑う。SEED を別の数字にしてみる
  • エージェントの出力がコードにならない → エージェントの指示に「出力は配列リテラルのみ。マークダウンのコードフェンスや説明文は一切含めない」を追加
  • ファイルを保存したのに反映されない → ブラウザのキャッシュ。Ctrl+Shift+R でハードリロード
